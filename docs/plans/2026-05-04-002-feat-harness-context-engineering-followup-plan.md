# Harness Overhaul — Followup (U4–U7 of 2026-04-27-003)

## Overview

Followup that closes out U4, U5, U6, and U7 from
`docs/plans/2026-04-27-003-feat-harness-context-engineering-plan.md`. The foundation work
(U1 source-of-truth + generators + validator, U2 MCP context-shaping, U3 tool consolidation
and autogenerated tool reference) shipped earlier on the predecessor plan. This plan
re-numbers the remaining four units as Units 1–4 and scopes Unit 3 explicitly to the
unfinished slice of the original U6 — the `.codex/agents/*.toml` files already exist; what
remains is `AGENTS.md`, the CLAUDE.md/AGENTS.md sync script, and the parity tests.

## Problem Frame

The predecessor framed three pressures: context-rot is not engineered against, skills and
agents under-use runtime extension points, and Codex is treated as second-class. U1–U3
addressed the foundational layer:

- **U1 — Source-of-truth, generators, validator** shipped in `c5d3e17`.
- **U2 — MCP context-shaping** (`outputSchema`/`structuredContent`, `response_shape`,
  `token_budget`, cursors, `cleanable`) shipped in `5afcd5c`.
- **U3 — Tool consolidation + autogenerated tool reference** shipped in `b23ea69`.

The remaining work is the visible end of the rewrite: skills still carry pre-U2 prose,
agents still pass `model: inherit` and the wide `Read, Grep, Glob, Bash` toolset, Codex
parity is half-complete (TOML files exist; `AGENTS.md`, sync script, and parity tests do
not), and the memory/handoff/telemetry plumbing that gives the harness its measurable
context-rot defenses is unbuilt. Without these four units, the foundational work in U1–U3
underdelivers — the new `cleanable` hint has no consumer, the new tool-reference layout has
no tightened skill spines pointing at it, and the regenerated agent .md files still show
the pre-rewrite tool surface.

## Scope Boundaries

**In scope (this plan):** the four predecessor units U4 (skill restructuring), U5 (agent
specialization), U6 *finish* (the unfinished slice — see Unit 3 below), and U7
(memory + handoff + observability + smoke tests). All `_source/`-driven content edits flow
through the existing generators landed in U1.

**Out of scope (already shipped — do not re-implement):**

- U1 — `plugins/kitty/_source/` tree, `scripts/generate_agents.py`,
  `scripts/generate_manifests.py`, `scripts/validate_skills.py`, pre-commit wiring.
  Implemented in `c5d3e17`.
- U2 — `outputSchema`/`structuredContent`, `response_shape`, `token_budget`, cursor
  pagination, `cleanable: true` hints across the MCP query/analysis/index/reactive tool
  surfaces. Implemented in `5afcd5c`.
- U3 — Shared `_traverse` for `find_dependencies`/`find_dependents` and the autogenerated
  per-family tool reference under `plugins/kitty/skills/kitty/references/tool-reference/`.
  Implemented in `b23ea69`.
- U6 *partial* — The nine `plugins/kitty/.codex/agents/*.toml` files already exist
  (cartographing-kitten + 4 librarians + 4 experts). Unit 3 below scopes only the
  remaining slice.

This plan also does not extend the graph (owned by `2026-04-23-001-plugin-evolution`) or
extract skills into a submodule (owned by `2026-04-27-001-skills-submodule-repository`).

    run_ids, not the full prose.
  - Happy: `query_litter_box(token_budget=2000, search="auth")` returns ≤2,000 tokens
    with a populated `truncated_items`.
  - Edge: a long run that triggers compaction retains the plan reference and handoff
    records; loses raw `cleanable: true` `index_codebase` payloads.
  - Edge: invoking `kitty:review` on a synthetic 12,000-token subgraph context spawns
    `expert-kitten-context` and produces a finding; the same skill on a 1,000-token

  - Error: deleting the graph and invoking `kitty:explore` triggers the SessionStart hook

## Risks & Dependencies

- **Risk: skill edits gated on submodule PR latency.** Mitigation: Unit 1 work proceeds
  on a feature branch in the submodule; the pointer bump is a separate cleanup once
  reviewers approve. Concurrent submodule PRs are batched.
- **Risk: token-budget approximations diverge from real Anthropic tokenization.**
  Mitigation: U2 already shipped the `tiktoken` cl100k_base path with a `len(json) / 4`
  fallback (5afcd5c). Unit 4's smoke tests rely on the same approximation.
- **Risk: skill spine truncation drops the Contract section.** Mitigation: enforce U1's
  spine order — frontmatter, then **decision tree → tool-selection matrix → Contract** as
  the first three H2s, then "References". Stage walkthroughs and templates live in
  `references/`. The validator (already shipped) asserts the order so a Contract section
  that drifts past line 5,000 fails CI.
- **Risk: agent-handoff TTL of 24h is too short for long-running plans.** Mitigation: TTL
  is per-record; plans that span days set `expires_at` to `created_at + 7d`. Default kept
  at 24h; orchestrator overrides as needed.
- **Risk: pre-U2 baseline cannot be recovered cleanly.** Mitigation: Unit 4's
  baseline-capture job cherry-picks telemetry instrumentation onto the pre-`5afcd5c`
  commit, records the medians, and stores them in
  `tests/fixtures/baseline_telemetry.jsonl`. If the cherry-pick is non-trivial, fall
  back to a synthetic baseline derived from the largest pre-U2 fixture run available in
  CI logs and document the substitution.
- **Dependency: `2026-04-27-001-skills-submodule-repository`.** Skill edits in Unit 1 land
  via the submodule. The `_source/` and `scripts/` directories stay in this repo per the
  existing repo-boundaries decision.
- **Dependency: `2026-04-23-001-plugin-evolution`.** When that roadmap adds
  hybrid-retrieval scores or LSP tooling, the new fields must fit the
  `compact`/`standard`/`full` shapes already shipped in U2 (`5afcd5c`).
